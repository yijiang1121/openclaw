OpenClaw

Personal AI assistant for your devices

OpenClaw is a local first assistant that lives on your hardware, responds on the channels you already use, and keeps control in your hands.

Gateway control plane Multi channel inbox Voice and Canvas CLI first workflows
Problem

Personal AI should feel local and always on

  • Messaging assistants are split across apps and devices.
  • Self hosted gateways are hard to keep running and secure.
  • Teams need a safe single user mode with allowlists and pairing.

OpenClaw focus

A single control plane that keeps sessions, tools, and channels in sync while running on your hardware.

How it works

Gateway connects everything

The Gateway is the heart of the system, hosting sessions, tools, and routing for every channel and device.

Gateway WebSocket control plane for sessions, tools, and events.
Pi agent Local runtime with tool streaming and session isolation.
Clients CLI, WebChat, macOS app, and mobile nodes connect on demand.
Reach

Built in and extension channels

Meet users on their preferred surface without rewriting workflow logic.

Core channels WhatsApp, Telegram, Slack, Discord, Google Chat, Signal, iMessage, WebChat.
Extensions Microsoft Teams, Matrix, Zalo, Zalo Personal, BlueBubbles.
Voice and nodes macOS, iOS, Android nodes with talk mode and Voice Wake.
Capabilities

Assistants that can act

  • Live Canvas for collaborative visual state.
  • Browser control and snapshots.
  • Media pipeline for images, audio, video, and transcription hooks.
  • Skills platform with bundled and workspace skills.
Safety

Guardrails by default

Pairing first Unknown DMs get pairing codes before any processing.
Allowlists Explicit allowlists per channel and account.
Doctor tools Built in checks to highlight risky settings.
Operations

Designed for reliable daily use

  • CLI onboarding wizard for setup across macOS, Linux, and WSL.
  • Install via npm, pnpm, or bun with Node 22 runtime.
  • Control UI and WebChat served directly from the Gateway.
